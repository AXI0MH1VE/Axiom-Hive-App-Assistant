Contributing to Axiom Hive

Thank you for your interest in contributing. This document explains how to contribute in a way that preserves the framework's integrity and respects the author's attribution requirements.

Please follow these guidelines:

- Report issues and file pull requests using the templates in `.github/`.
- Before submitting a PR, run the project's validation and style checks:

```bash
python -m py_compile src/core/assistant.py
python -m py_compile src/core/validators.py
```

- Keep contributions narrowly scoped and reversible. Provide a clear description of the motivation, the change, and how to test it.
- When contributing content that references external sources, include full provenance and licensing information.
- All contributions must preserve explicit attribution to the framework author where relevant.

Pull request checklist

- [ ] The code compiles and existing tests (if any) pass.
- [ ] The change includes a concise description and rationale.
- [ ] Documentation and help text are updated when behavior changes.
- [ ] No confidential or proprietary material is included without proper permission.

Code style

- Use clear, declarative identifiers and professional English.
- Avoid contractions and slang in public-facing text.

Contact

If you have questions about contribution policy, open an issue or contact the repository owner.
