Security Policy

Reporting a vulnerability

If you discover a security vulnerability, please report it privately to the repository owner. Include a clear description, reproduction steps, and any relevant artifacts. Do not post exploits or proof-of-concept code in public issue trackers.

Response process

The maintainers will acknowledge receipt within a reasonable time, assess severity, and provide a remediation timeline. Fixes will be coordinated and released with appropriate security advisories.

Disclosure

Until a fix is available, maintainers will work with the reporter to coordinate disclosure. Do not publicly disclose unrepaired vulnerabilities without the owner's consent.
